# DECISIONS — PL (project-specific)

_Standing decisions for Programming Lab. Cross-lab decisions live in `HQ/DECISIONS.md` (cite by code, e.g. D-07). Reference these by code, don't re-explain._

- **D-PL-24 — Foundations driven models are a config-driven template where the shape repeats; bespoke only where the picture is the lesson.** The KNOW interactive models must not be 73 hand-written React components (that is the PAL-scale trap — interactivity = code does not scale). The recurring "binding & identity" shape (drive ops → build Python → run live in Pyodide → render values + an `is` verdict) is **one template, `StateTrace.jsx`, configured by DATA** (`foundationsModels.js`): a new such model is a config entry, not a component. Aliasing + copy-vs-view collapsed onto it; mutable-default added as config-only (proof: a new driven module = data). Feasible only because PL has a real Python runtime to be the engine — PAL can't (no runtime), which is why PL's interactivity can be declarative where PAL's must be bespoke SVG. **Bespoke widgets stay** only where the dynamics/picture genuinely need custom viz (call stack, Big-O, hashing, broadcasting, the numpy race). Net: filling the rooms is mostly authoring, not engineering. (Sidharth: "your interactiveness means code, no" — yes, so the repeating ones become data.)
- **D-PL-23 — Dark mode is Graphite (Platinum at night), not a separate skin.** PL has **one skin — Platinum** — with **two themes**: light (the adopted default, shared across labs) and **dark = Graphite** (charcoal desktop, raised graphite cards, same beveled chrome + Geneva + red seam). Implemented as `:root[data-skin='platinum'][data-theme='dark']` (`index.css`); the footer toggle flips `data-theme`, skin stays Platinum so the menu bar / bevels / text-only nav carry across both. **Supersedes D-PL-18** (the green-screen P1-phosphor CRT identity): the green-screen is retired as the dark mode — `skin.js` coerces any stale stored skin to platinum, so green is unreachable (its tokens/overlay stay dormant in the stylesheet pending cleanup). Chosen from a 3-up mockup (Graphite / Blue Graphite / Inkwell); Sidharth picked Graphite — "get rid of the black-green thing." Approved 2026-06-24.
- **D-PL-22 — PyLab architecture: the two-axis judgment gym.** PyLab is positioned as the "Python judgment gym" — the white space between LeetCode (algorithms, output-graded) and StrataScratch/DataLemur (SQL/pandas, output-graded): it grades **judgment** (method fork + the runs-but-wrong trap + cost + craft), the thing every role/level interview actually tests ("runs vs right"). Every problem is tagged on **two axes — ROLE** (SWE/DS/MLE/AIE/DA/BA/PA) **× SENIORITY** (fluency→correctness→judgment→systems), derived/override-able (`pyLabMeta.js`). The runner is a platform for **9 formats** over the same gated content (Predict/Solve/Forensic + Refactor/Code-review/Ambiguity-drill/Scale-it/Follow-up/Mock-loop/Take-home/Explain-it) — author depth once, surface it five ways. Scope holds (D-PL-15/D-PL-07): programming + data craft, not model/LLM theory; easy→medium DSA ceiling. **Full blueprint + phased roadmap: `docs/PYLAB-VISION.md`** (from market research across all seven roles, 2026-06-24). Phase 1 (role/level + readiness) shipped 0.20.0.

- **D-PL-21 — KNOW becomes the Foundations rooms (trunk + branches); two branches amend the charter.** PL's KNOW frame is re-scoped from the 20-card `knowModules.js` stub into a room architecture mirroring PAL's Foundation rooms — **5 trunk rooms** (Python Foundations · The Machine · Data Structures & Algorithms · NumPy & pandas · Shipping Python) every learner climbs, **+ 2 branches** (Competitive Programming · Tensors & Autograd) that diverge after. Triggered by `docs/FOUNDATIONS-HANDOFF.md` (PAL's mentor note: *teach by manipulation, not exposition*) + a live read proving PAL's KNOW is ~10x PL's whole codebase and genuinely slider-driven where PL's is predict-run-read. **Bar:** every module centers a *manipulable model the learner drives*, not a fixed snippet + MCQ. **Build (not a rebuild):** reuse the existing `KnowRunner` + add one optional `interactive` widget slot (between demo and reveal; back-compatible like `normalizePredict`, D-PL-17); a registry-driven `FoundationsBrowser` reads `src/data/foundationsRooms.js` (the skeleton — 7 rooms / 24 clusters / 73 seed modules, status `planned`). Four widget substrates: **`live`** (real Pyodide run — PL's edge over PAL, *prefer it*), **`sim`** (bespoke SVG where the picture is the lesson), **`stepper`** (state machine where the dynamics are), **`concept`** (explainer-only, never faked). **Charter amendment (the deliberate part):** Branch 6 (Competitive Programming) is consciously **above the easy->medium ceiling** — partially supersedes **D-PL-07**, which still governs the *trunk* (Room 3 DSA stays easy->med); Branch 7 (Tensors & Autograd) takes **library mechanics only** (tensor/autograd/broadcasting/shapes) — modeling/training stays in `ml-systems-lab`, the same KNOW->DO seam. Re-scopes PL from "fluency floor" to "fluency floor + two depth verticals"; if a branch ever starves the trunk or conflicts with the distribution gate (D-PL-06), the branch pauses. Approved 2026-06-24. **Authority spec: `docs/FOUNDATIONS-SPEC.md`** (room scope, through-lines, build order F0-F7).
- **D-PL-20 — PyLab built to the SQL-Lab bar (judgment gym, not a grader).** PyLab is the single entry for **pandas/numpy AND Python** (`docs/PYLAB-HANDOFF.md` + `docs/PYLAB-BUILD-SPEC.md`). Contract (locked, sign-off 2026-06-24): `solve(...)→output` graded by the typed comparator `pl_compare` (DataFrame/Series/array/value, explicit dtype/index/column-order/NaN) — **retires `__pl_checks` for PyLab content**. Engineered seed DataFrame fixtures (`pyLabFixtures.js`); per-problem judgment layer (`methods[]`/`dial`/`mcqs`) with `JudgmentLayer.jsx`. **Four committed gates run before commit:** `audit_py.py` (Tier-1 0), `verify_py_methods.py` (non-trap == solution, trap runs+diverges, id integrity), `py_content_scan.mjs` (jargon/hints/debrief), `run_py.py` (`--diverge` authoring). Non-negotiables: **execute everything** (never hand-write an expected value or fake a trap), de-jargon prompts, **honesty rule** (single-method → empty dial), gates before every commit. Foundation + 4-problem seed shipped (0.17.0), all CPython-verified; banks authored on top in gated batches.
- **D-PL-19 — Pluggable SKIN system; Platinum is the active look (supersedes the green-screen as default, D-PL-18).** A "skin" is PL's whole visual world (chrome + palette + fonts), applied via `[data-skin]` on `<html>` and swapped in one line (`setSkin()`), orthogonal to the dark/light toggle. Skins: **`platinum`** (classic Mac OS 8/9 workstation — ACTIVE default, Apple menu bar + rainbow apple + beveled Platinum), **`greenscreen`** (the P1 CRT terminal, now a secondary skin that will live inside terminal windows), **`aqua`** + **`hybrid`** (reserved — the Mac OS X terminal and the Platinum-desktop-opens-Aqua-terminal model). Mechanism: `src/utils/skin.js` + anti-flash in `index.html` + `:root[data-skin='…']` token blocks appended last in `index.css` (so the active skin overrides) + React-gated chrome (`PlatinumMenuBar`). A skin must never touch problem data / runner / progress — look only. Built incrementally (unit 1 = Platinum skin + system; the Finder-window + terminal interaction is the next unit). **Governing spec: `docs/SKIN-SYSTEM.md`.** (Sidharth: "any design choices we have right now should not supersede the design choices here... make it switchable so changing it won't take a heavy build again.") **Refined (0.16.0):** **light = Platinum (the only light mode; casefile retired), dark = green-screen now / Aqua next.** Real Platinum = white content cards on a gray window frame, crisp black borders, Chicago system font (cdnfonts), no colored side rails (all 16 stripped). Footer toggle = one sun/moon swapping light(Platinum)⇄dark. Dark mode is being reframed to **Mac OS X Aqua + ambient Matrix rain** (green becomes the terminal *content*; rain dim behind work, full only on load/reveal/idle) — light=classic Mac, dark=OS X, two Apple eras. Lesson logged: a token-recolor is NOT a skin — a skin must rebuild to the referenced design's actual language (white panes, bevels, scrollbars, system font), superseding our own patterns.
- **D-PL-18 — Identity = the green screen (old-school CRT), dark mode only. Supersedes D-PL-04 (violet "Instrument"). [Now a SKIN under D-PL-19, not the default look.]** Each BreakLabs lab's dark mode is its single defining color; light mode is shared family "Field Notes" (untouched). PL's color is **P1 phosphor green `#46E08A`** on a near-black void — the open old-school slot (P3 amber = MSL, blue = PAL, teal = GSL) and the literal display the base layer boots into. The identity is carried by a *treatment*, not just hue: scanlines + vignette overlay, phosphor text-glow, monospace chrome, inverse-video active nav (TUI-installer highlight), box-drawing legend panels (`pl-panel`), and the HUD glass-box readout (`pl-hud`). Palette logic: green = baseline/OK, red = break, amber = signal. Body text is soft greenish-white for readability. The red brand seam stays constant across labs (HQ D-19); only the descriptor goes green. (Earlier violet was a flat modern UI tinted — rejected as "not old-school"; the win was the CRT surface treatment, not the color alone.) **Refined (0.13.0): dark mode is green + black ONLY** (every semantic hue collapses to a green tier; meaning = brightness + glyphs + inverse, never hue), the **period terminal font VT323** (no curved lettering) governs the whole dark UI (light keeps Inter), and **no white lettering on green fills** (dark phosphor ink only). **Finalized (0.14.0):** font = **Courier Prime** (VT323 retired as ugly); **all text phosphor green on pure-black** (`--bg #000000`, body `#4FE08C`, bright headings `#7FF5B0`, no white); **no card highlights/decoration** (law #7 — top-accent bars + hover glow removed across all browsers). **Full governing spec — the authority for any PL visual change — is `docs/GREEN-SCREEN-IDENTITY.md`.**
