# DECISIONS — PL (project-specific)

_Standing decisions for Programming Lab. Cross-lab decisions live in `HQ/DECISIONS.md` (cite by code, e.g. D-07). Reference these by code, don't re-explain._

- **D-PL-01 — Option A architecture.** PL is a React + Vite + Pyodide SPA, sibling-consistent with PAL/MSL/GAL. The Dec-2024 FastAPI/Docker infra lab was archived to `_legacy/`, not revived. (Rejected: server-side code execution — a security/cost/ops liability the other labs avoid.)
- **D-PL-02 — Reuse, don't reinvent.** PL adopts sibling components (MSL `PythonCell`, PAL `Sidebar`/`Icon`/`ForwardPointerCard`, MSL `HowToStrip`, synthesized `GateOverlay`, PAL `unlock`). One Pyodide loader, one component system across labs. (HQ DESIGN-STANDARD.)
- **D-PL-03 — Glass-box is the moat, built early.** The time/memory cost view (and the future judgment dial) ship with the IDE, not last. A pass/fail runner is commodity; the cost view is not.
- **D-PL-04 — Identity = Instrument (dark-first).** PL takes its assigned BreakLabs track colour **violet `#8B5CF6`** on void, mono-forward (Inter + JetBrains Mono), with gold/red/green master semantics. Light "Field Notes" mode via `[data-theme]`. PL is the lab the "Glass Box" brand was describing, so it earns the dark instrument register. (Rejected: Terminal — too cold; Daylight — too close to PAL.)
- **D-PL-05 — Slug renamed, folder kept.** GitHub repo is `programming-lab`; the local folder stays `labs/production-systems-lab` so the Cowork mount + `BreakLabs/CLAUDE.md` paths keep working. Rename the local dir only in a dedicated infra pass.
- **D-PL-06 — Distribution-gate override is conditioned.** PL was held behind the distribution keystone (D-01); the override holds *only* because every Bank-A gotcha is also a LinkedIn post. If the daily slips, PL pauses.
- **D-PL-07 — Easy→medium ceiling.** No contest-grade DSA (no segment trees, heavy DP, contest math). PL is the fluency floor, not a competitive-programming grind (D-07).
- **D-PL-18 — Identity = the green screen (old-school CRT), dark mode only. Supersedes D-PL-04 (violet "Instrument").** Each BreakLabs lab's dark mode is its single defining color; light mode is shared family "Field Notes" (untouched). PL's color is **P1 phosphor green `#46E08A`** on a near-black void — the open old-school slot (P3 amber = MSL, blue = PAL, teal = GSL) and the literal display the base layer boots into. The identity is carried by a *treatment*, not just hue: scanlines + vignette overlay, phosphor text-glow, monospace chrome, inverse-video active nav (TUI-installer highlight), box-drawing legend panels (`pl-panel`), and the HUD glass-box readout (`pl-hud`). Palette logic: green = baseline/OK, red = break, amber = signal. Body text is soft greenish-white for readability. The red brand seam stays constant across labs (HQ D-19); only the descriptor goes green. (Earlier violet was a flat modern UI tinted — rejected as "not old-school"; the win was the CRT surface treatment, not the color alone.) **Refined (0.13.0): dark mode is green + black ONLY** (every semantic hue collapses to a green tier; meaning = brightness + glyphs + inverse, never hue), the **period terminal font VT323** (no curved lettering) governs the whole dark UI (light keeps Inter), and **no white lettering on green fills** (dark phosphor ink only). **Finalized (0.14.0):** font = **Courier Prime** (VT323 retired as ugly); **all text phosphor green on pure-black** (`--bg #000000`, body `#4FE08C`, bright headings `#7FF5B0`, no white); **no card highlights/decoration** (law #7 — top-accent bars + hover glow removed across all browsers). **Full governing spec — the authority for any PL visual change — is `docs/GREEN-SCREEN-IDENTITY.md`.**
- **D-PL-17 — KNOW adopts PAL's Stats-Room Foundations format.** KNOW modules carry leveled predict options (`{label, level:'strong'|'partial'|'wrong', feedback}`) + a `seniorRead` block (`shortAnswer / why / commonMistake / interviewPhrase / connectsTo`), rendered by `SeniorRead.jsx` (a port of PAL's `StatsConceptPanel`). The model is PAL's **Stats Room** (data-driven), NOT the literal "Foundations" rooms (hardcoded-per-module JSX — studied and rejected as unscalable). PL keeps its differentiator on top: the runnable Pyodide demo. `KnowRunner.normalizePredict` stays back-compatible with the legacy `answerIndex` shape so future banks can migrate incrementally.
- **D-PL-16 — Single bank registry + Progress dashboard.** `src/data/banks.js` is the one source of truth for every bank (frame, view, total, progress key, accent). The Progress TRACK page aggregates per-bank localStorage progress into a completion + readiness-by-frame view (PAL's Progress room, adapted). Register new banks there.
- **D-PL-15 — PL is the programming/Python-depth lab, not GenAI.** Scope stays on coding, Python, CS, DSA, pandas, OOP, and SWE craft (testing/typing/idioms). The JUDGE "Spot the Flaw" frame drills CODE correctness from real Python/pandas anti-patterns; that buggy code is increasingly LLM-generated is incidental *motivation*, never the identity — GenAI is GSL's lane (Sidharth's call, 2026-06-23). Build programming-first; sequence content in verified passes, no grind.
- **D-PL-14 — All four Competence-Model frames carry real content.** KNOW (runnable depth explainers), DO (the drill banks), BUILD (scaffolded stepper projects), JUDGE (forensic "code that runs and lies" — PL's judgment layer, the analog of SQL Lab's Forensic tier). A shared `DebriefBlocks` (marker→collapsible) renders debriefs across frames. Built from a deep SQL Lab study; no frame ships as a SOON stub once it has ≥a few verified entries.
- **D-PL-13 — Two-pane solve view; expected output computed live from the solution.** The drill UI mirrors PAL's SQL Lab: prompt + data-preview + expected-output panels | editor + Run/Submit + timer + targeted feedback. Each problem carries a verified `example {setup, call, inputs}`; the runtime `previewExample` runs it against the canonical solution and renders the result — the expected answer is never hand-maintained (matches the audit's "solution is source of truth"). Browse is a filterable table (difficulty/solved/search), not a card grid.
- **D-PL-12 — Committed audit gate, the SQL-Lab way.** Every commit touching problem data must pass `scripts/audit_problems.py` (0 Tier-1). The solution is the source of truth, verified against its own `__pl_checks` — never hand-maintain expected outputs (PAL's principle). Inherited from a study of PAL's SQL Lab. What does NOT carry: sql.js / `CREATE TABLE`, `expectedColumns/rowCount` as the contract, `ORDER BY` determinism + float-normalization hacks — PL uses Pyodide + pandas-native NaN-aware comparison + an AST sandbox instead. Bar: `docs/CONTENT-STANDARD.md`.
- **D-PL-11 — Test-based problem schema + runner.** Drill banks (Python, pandas) use `{ prompt, starterCode, testSource (defines __pl_checks), solution, glassBox.lesson }`. The Pyodide runtime runs the solution + checks and reports per-check pass/fail + cost; pandas/numpy wheels lazy-load on demand. Every solution + test is verified in real CPython/pandas before shipping. (Distinct from the gotcha predict→reveal shape; both reuse `PythonCell` + the glass-box.)
- **D-PL-10 — Adopt the HQ Sidebar Standard (DESIGN-STANDARD.md).** PL's left nav is the four-frame accordion (KNOW/DO/BUILD/JUDGE), one-open-per-level, measured-height motion, frame icons, violet active pill, SOON badges (PL's own pattern). The `⌘K` search slot is deferred while PL has a single live tab (nothing to search yet) — add it when banks multiply (A-PL-05).
- **D-PL-09 — Adopt the BreakLabs BrandMark (HQ D-19).** PL renders the canonical `break⌇labs` lockup (red seam + wordmark constant across all labs) with descriptor `Programming` in violet. The favicon is the shared red monogram — identical across labs by design (the unity; D-14 repetition). Built from the `HQ/BRANDMARK-ROLLOUT.md` reference, wired into the 7 standard slots.
- **D-PL-08 — Spine at the lab root.** PL's operational spine (CLAUDE, README, CHANGELOG, STATUS, NEXT, LINEAGE, DECISIONS, AUDITS, IDEAS) lives at the lab folder root, matching the siblings; only governing specs (`PL-BUILD-SPEC.md`) live in `docs/`. (Diverges from HQ `PROTOCOL.md`, which says operational state goes in `docs/` — the siblings predate that protocol; flagged for HQ to reconcile. Sidharth's call.)
